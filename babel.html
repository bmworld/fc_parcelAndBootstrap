BABEL- 구형브라우저에 최신JS문법으로 Compile

(ES6 ~ ES8) -> ES5

$ npm i -D @babel/core @babel/preset-env

바벨 설정파일 생성 (.babelrc.js)

module.exports =
{ presets: ['@babel/preset-env']}

설정완료 => 이제 모든 Javascript코드는 babel을 통해서 ES5문법으로 변환된다.

Parcel 은 모듈 안에서 설정 파일(예: .babelrc, .postcssrc)을 발견했을 때 자동으로 이 변환을 실행합니다.

Babel은 package.js 속에 browerslist에 영향을 받아 브라우저마다 최적화하여 변환한다. ==> 필수 입력하시라

Babel 설치 후에도, 특정 JS문법 사용 시 Error가 발생할 수 있다. 그렇다면 추가적ㅇ린 플로그인을 설치해야한다.

$ npm i -D @babel/plugin-transform-runtime

module.exports =
{ presets: ['@babel/preset-env']},
plugins: ['@babel/plugin-transform-runtime']

Bundler는 개발용과 배포용을 나누어서 설정해줘야 한다.

링크를 참조하여, 옵션이나 포트 등을 설정할 수 있다.

https://ko.parceljs.org/cli.html

    옵션: 기본값은 dist폴더에 빌드가 된다.
  • $ parcel build entry.js --out-dir [folderName]
  • 빌드되는 [폴더명]을 변경할 수 있다.
  • html, scss, css, js... 어떤 파일이든 모두 build대상이 된다.
  • 빌드파일: html --> HTML에 구동되기에 필요한 모든 파일(css, js, scss)이 build대상이 된다.
  • 빌드파일: css --> css파일만 build된다.
  • ex. $ parcel build ./scss/main.scss --out-dir build
  • root/build폴더에 경로에 해당하는 main.scss파일만 Build되어 main.css파일, main.css.map파일로 compile된다.
  • $ parcel babel.html --port 8088 --open
  • babel.html파일을 로컬에서 실행하며, 포트번호는 8088로 , 브라우저에서 즉시 열리도록(--open) 한다.
